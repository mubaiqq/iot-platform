# v1.2.4 轻量生产加固说明

发布日期：2026-07-14

本版本面向个人使用和小规模公网分享，重点修复会直接影响账号、数据和设备控制的安全问题，同时避免采用会频繁误拦正常用户的重型策略。

## 安全与权限

- 公共设置接口仅允许读取注册开关和验证码开关，不再暴露 MQTT 密码、天气 API Key 等敏感配置。
- 设备命令、命令确认、设备日志和待绑定设备删除均校验设备归属；管理员保留明确的管理权限。
- MQTT 在线测试会话使用高熵随机 ID 并绑定当前用户，普通用户只可操作自己设备的 Topic；Broker 地址仍可按正常业务公开使用。
- LLM 配置列表不再向浏览器返回完整 API Key；编辑时留空即可保留原 Key。
- LLM 外部请求拦截明显的本机/私网目标并禁止重定向，正常公网和自建公网 API 不受影响。
- 修改用户名或密码后撤销旧登录 Token，并提供真正的“退出所有设备”。
- 增加基础安全响应头、请求体大小限制、轻量同源检查，以及宽松的登录/注册防刷阈值。
- 全新安装不再使用公开固定的默认管理员密码，由安装流程初始化管理员凭据。

## 前端与稳定性

- 修复设备日志、设备详情、天气结果和 LLM 配置页中不安全的动态 HTML 渲染。
- 原生确认框替换为站内确认组件，保持交互风格一致。
- 手动浇水会等待 MQTT 发布结果，Broker 未连接时不再误报“发送成功”。
- 修复旧版管理框架脚本中的行号污染，恢复 JavaScript 语法有效性。
- 宿主机更新脚本在 PM2 重启前自动运行测试，测试失败即停止更新。
- Docker 应用改用非 root 用户运行。

## 依赖与测试

- Playwright 更新至 1.61.1。
- 新增后端安全、前端渲染、安装流程和运维更新守卫测试。
- 生产及开发依赖执行 `npm audit`，目标为零已知漏洞。

## 兼容性说明

- 现有数据库、账号、设备和固件无需重建。
- MQTT Broker 地址、端口及正常调试能力保持不变。
- 本版本没有修改 ESP32 通信协议，因此固件版本仍保持 v1.2.3；平台应用版本升级为 v1.2.4。
