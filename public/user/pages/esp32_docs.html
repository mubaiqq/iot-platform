1 概述

木白云 IoT 平台使用 MQTT 协议与 ESP32 设备通信。设备固件基于 Arduino 框架，支持 AP 自助配网、MQTT 自动注册、远程浇水控制、定时计划任务等功能。

核心特性：
✅ AP 热点配网（无需写死 WiFi）
✅ MQTT 自动注册设备码
✅ 心跳上报 + 传感器数据
✅ 远程浇水控制（手动 / 定时 / 智能AI）
✅ 计划任务本地执行 + NTP 校时
✅ 断线自动重连 + 闪存持久化
ESP32 设备
↕ WiFi
MQTT Broker（mqtt.mcoud.cn）
↕ 订阅/发布
木白云 IoT 平台
↕ 浏览器
用户控制台
2 硬件准备
组件说明备注
ESP32 开发板任意 ESP32-WROOM / ESP32-S3推荐 ESP32-DevKitC
继电器模块5V 单路继电器默认 GPIO 26
电磁阀 / 水泵12V 常闭电磁阀通过继电器控制
电源5V 2A USB 供电建议带稳压
（可选）土壤湿度传感器电容式土壤湿度接 ADC 引脚
（可选）DHT11 温湿度温湿度传感器接任意 GPIO
接线注意： 继电器控制端接 ESP32 的 GPIO 26，继电器 COM 端接电源正极，NO 端接电磁阀正极，电磁阀负极接电源负极。请确认继电器是 低电平触发 还是 高电平触发，代码默认 高电平触发（HIGH = 开启）。
3 软件环境
1

安装 Arduino IDE

下载 Arduino IDE 2.x，安装完成后打开。

2

添加 ESP32 开发板支持

打开 文件 → 首选项 → 附加开发板管理器网址，填入：

开发板 URL
https://espressif.github.io/arduino-esp32/package_esp32_index.json

然后打开 工具 → 开发板 → 开发板管理器，搜索 esp32 并安装。

3

安装依赖库

打开 工具 → 管理库，搜索并安装：

库名用途
PubSubClientMQTT 客户端
ArduinoJsonJSON 解析（选 v6.x）

WiFi.h / WebServer.h / Preferences.h 为 ESP32 SDK 自带，无需额外安装。

5 配网流程

ESP32 首次启动（或 WiFi 连接失败时）会自动进入 AP 配网模式：

ESP32 首次上电
↓ 自动
开启热点「木白云IoT-配网」
↓ 手机连接
自动弹出配网页面
↓ 选择 WiFi + 填写信息
保存并重启
↓ 自动
连接 WiFi + MQTT 注册
配网页面说明：
• WiFi 网络：自动扫描周围 WiFi，点击选择
• WiFi 密码：输入所选 WiFi 的密码
• MQTT 服务器：默认已填好 mqtt.mcoud.cn，一般无需修改
• 用户名（选填）：填写后设备会自动关联到你的平台账号
重新配网： 如果需要更换 WiFi，将 ESP32 重新上电或长按重置按钮，WiFi 连接失败 30 秒后会自动重新进入配网模式。
6 MQTT 协议

设备通过 MQTT 与平台通信，以下是完整的 Topic 规范：

Topic方向说明
device/{code}/register 发布 首次连接时注册设备
device/{code}/heartbeat 发布 每 30 秒上报心跳 + 传感器数据
device/{code}/status 发布 浇水确认 / 计划任务确认
device/{code}/watering_complete 发布 浇水完成后上报执行结果
device/{code}/watering_request 发布 智能浇水请求（等待 AI 判断）
device/{code}/command 订阅 接收平台下发的控制指令

📡 注册消息格式

device/{code}/register
{
  "device_code": "Kxxxxxxxxxx",
  "username": "your_username"
}

💓 心跳消息格式

device/{code}/heartbeat
{
  "device_code": "Kxxxxxxxxxx",
  "timestamp": "",
  "sensor_data": {
    "soil_moisture": 65
  },
  "schedule_count": 2,
  "current_time": "14:30"
}
7 心跳机制

设备每 30 秒 向平台发送一次心跳，平台通过心跳判断设备是否在线。

心跳内容包含：
• 设备码（device_code）
• 传感器数据（如有接传感器）
• 当前计划任务数量
• 设备本地时间
超时判定： 平台如果 90 秒（3 个心跳周期）未收到心跳，会将设备标记为离线。
8 浇水控制

平台通过 MQTT 下发浇水指令到设备：

🟢 开启浇水

平台 → 设备 command
{
  "water": true,
  "duration": 30
}

duration 单位为秒，范围 1-600，默认 30 秒。

🔴 停止浇水

平台 → 设备 command
{
  "water": false
}

✅ 设备回复

设备 → 平台 status
{
  "success": true,
  "water": true,
  "duration": 30
}

📋 执行结果上报

设备 → 平台 watering_complete
{
  "timestamp": "",
  "duration": 30,
  "source": "manual"
}

source 取值：manual（手动）、schedule（计划任务）、schedule_ai_failed（AI 判断失败）

9 计划任务

平台可以向设备同步定时浇水计划，设备通过 NTP 自动校时，到点自动执行。

📅 平台下发计划任务

平台 → 设备 command
{
  "command": "schedule",
  "tasks": [
    {
      "time": "08:00",
      "duration": 60,
      "enabled": true,
      "fixed_watering": true
    },
    {
      "time": "18:00",
      "duration": 30,
      "enabled": true,
      "fixed_watering": false
    }
  ]
}
字段类型说明
timestring执行时间，格式 HH:MM（24小时制）
durationint浇水时长（秒）
enabledbool是否启用
fixed_wateringbooltrue=固定浇水，false=智能AI判断
智能浇水 vs 固定浇水：
• 固定浇水（fixed_watering=true）：到时间直接执行
• 智能浇水（fixed_watering=false）：到时间先请求 AI 判断，AI 回复 {"water":true} 才执行；超时 60 秒重试 1 次，共等 160 秒

11 常见问题

❓ 设备一直显示离线？

1. 检查 WiFi 信号是否稳定
2. 检查 MQTT 服务器地址是否正确（mqtt.mcoud.cn）
3. 打开 Arduino IDE 串口监视器（波特率 115200）查看日志
4. 如果显示 rc=-2，说明 MQTT 连接被拒绝，检查网络

❓ 如何重新配网？

方法 1：长按重置按钮 5 秒，清除闪存配置后重启
方法 2：将 ESP32 断电，在新 WiFi 环境下重新上电，连接失败会自动进入配网

❓ 继电器不动作？

1. 确认继电器触发方式（高/低电平），代码默认 HIGH 触发
2. 检查 GPIO 引脚号是否正确（默认 26）
3. 串口查看是否收到浇水指令

❓ 计划任务没有执行？

1. 确认 NTP 时间同步成功（串口会打印当前时间）
2. 确认计划任务已同步到设备（串口显示 [计划] 收到 X 个计划任务）
3. 同一时间点的任务每天只执行一次，不会重复触发
