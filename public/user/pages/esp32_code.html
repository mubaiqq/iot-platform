ESP32 固件代码 v2.1.2
复制以下 v2.1.2 代码到 Arduino IDE，烧录到 ESP32 即可使用；v2.1.2 已兼容旧控制器 GPIO16 继电器接线，增加 water=false 安全停泵，并且计划任务不补执行错过时间
使用说明：
1. 安装 Arduino IDE，添加 ESP32 开发板支持
2. 安装库：PubSubClient（MQTT）、ArduinoJson（v6）
3. 修改代码中的 MQTT服务器 地址
4. 烧录后首次启动会打开 AP 热点 木白云IoT-配网（密码 12345678），连接后自动弹出配网页面
5. 配网成功后自动通过MQTT注册设备码并连接平台

     需要修改的配置项：


    MQTT_SERVER — MQTT服务器地址，如 mqtt.mcoud.cn

    MQTT_PORT — MQTT端口，默认 1883

    RELAY_PIN — 继电器引脚，已按旧设备改为 GPIO 16

    RELAY_ACTIVE_LEVEL — 继电器开启电平，已按旧程序保持 HIGH 开启、LOW 关闭


    已兼容旧设备逻辑：

    • 旧程序的继电器引脚 GPIO16

    • 旧程序的最大浇水时长 600秒

    • 旧程序的智能/自动浇水冷却：浇水时长 × 3

    • 心跳附带 WiFi连接状态、设备IP、RSSI、继电器引脚、当前浇水状态

    • 计划任务只在设定时间所在分钟执行；设备重启后如果已错过时间，不会补执行


    可选配置：

    HEARTBEAT_INTERVAL — 心跳间隔(毫秒)，默认 30000 (30秒)

    SOIL_PIN — 土壤湿度传感器引脚 (ADC)

    DHT_PIN — DHT11温湿度传感器引脚


     计划任务功能：

    设备通过NTP自动同步时间，收到计划任务后保存到本地闪存。

    固定浇水：到时直接执行，完成后上报日志

    智能浇水：到时向平台请求AI判断，根据AI回复执行（60秒超时重试1次，共等160秒）


     恢复出厂：

    串口发送 FACTORY_RESET 即可清空配网信息，重新进入AP配网模式
  


  

     恢复出厂设置（重置配网）：


    在串口监视器中发送：FACTORY_RESET


    注意：

    • 波特率：115200

    • 串口监视器结尾建议选 新行 或 NL

    • 命令不区分大小写，代码里会自动转大写

    • 执行后会清空 NVS 配置、断开 WiFi、关闭继电器，然后自动重启

    • 重启后进入 AP 配网模式，AP 密码 12345678


    串口会看到：[factory] clearing NVS and restarting...，然后设备重启


    继电器测试命令：

    • RELAY_TEST：继电器开启3秒后自动关闭

    • RELAY_ON：安全开启30秒倒计时后自动关闭

    • RELAY_OFF：立即关闭继电器
