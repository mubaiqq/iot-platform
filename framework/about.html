技术架构

整体架构

采用 iframe 单页应用（SPA）模式。主框架 index.html 负责导航、标签管理、弹窗、消息提示等容器功能，子页面通过 iframe 加载，各自独立运行。

前端

原生 HTML/CSS/JS，无框架依赖
Font Awesome 6.5 图标
Inter 字体
CSS 变量统一主题色

后端

Node.js + Express
Session 认证
Nginx 反向代理
Ubuntu 服务器

容器 API（子页面调用）

子页面通过 parent 对象调用主框架功能：

方法参数说明
parent.openTab()id, title, icon, closable, url打开新标签页。id 为标签标识，icon 为 FA 图标类名（不含 fa-），url 可选（覆盖 PAGE_MAP）
parent.openModal()url, title, w, h打开弹窗。w/h 为宽度/高度（如 '600px'、'80vw'）
parent.openModalPost()url, title, params, w, hPOST 方式打开弹窗。params 为 {key: value} 对象
parent.closeModal()—关闭弹窗（带下滑动画）
parent.showToast()msg, type显示消息。type: success / error / info
parent.showConfirm()title, msg, opts自定义确认弹窗（Promise）。opts: type(danger/info), okText, cancelText, icon
parent.closeTab()id关闭指定标签

调用示例

// 打开一个新标签 parent.openTab('user_list', '用户列表', 'fa-user', true); // 打开一个非 closable 的标签（首页） parent.openTab('welcome', '首页', 'fa-house', false); // 打开带自定义 URL 的标签 parent.openTab('add_3', '添加-' + username, 'fa-plus', true, 'add_domain.html?useridid=' + username); // 打开弹窗 parent.openModal('edit_domain.html?id=1', '编辑域名', '600px', '70vh'); // POST 弹窗（表单提交） parent.openModalPost('stats.html', '域名统计', {id: 1, ym: 'example.com'}); // 显示成功提示 parent.showToast('操作成功', 'success'); // 显示错误提示 parent.showToast('提交失败', 'error'); // 确认弹窗（危险操作，红色按钮） parent.showConfirm('确认删除', '删除后无法恢复', { type: 'danger', okText: '删除' }).then(function(ok) { if (ok) { /* 执行删除 */ } }); // 确认弹窗（普通确认，蓝色按钮） parent.showConfirm('提示', '确定要这样做吗？').then(function(ok) { if (ok) { /* 执行操作 */ } });
标签页系统

标签生命周期

打开：调用 openTab() → 创建 iframe → 渲染标签栏 → 滚动到可见区域

切换：点击标签 / 侧边栏 → activateTab() → 切换 active 类

关闭：点击关闭按钮 / 右键菜单 → closeTab() → 移除 iframe 和标签

首页标签不可关闭（closable=false），始终显示。

标签操作

点击 切换标签

点击 ✕ 关闭标签

右键 上下文菜单

下拉按钮 关闭当前 / 其他 / 全部

自动滚动 新标签和切换时自动滚动到可见区域

弹窗系统

弹窗行为

打开动画：从底部滑入（translateY + scale）

关闭动画：向下滑出（closing 类 + 220ms 延迟）

桌面端：居中显示，宽度 max(80vw, 800px)，高度 max(75vh, 600px)

移动端：全屏底部弹出，圆角顶部 16px

关闭方式：点击关闭箭头 / 点击遮罩层 / ESC 键

openModal vs openModalPost

openModal() 直接 GET 加载 URL 到 iframe。

openModalPost() 在 iframe 内动态创建 form 并 POST 提交，适合传递隐藏参数（如 ID），URL 不暴露参数。

showConfirm 确认弹窗

替代浏览器原生 confirm()，返回 Promise，支持过渡动画（背景模糊 + 卡片缩放）。

type: danger 红色按钮，用于删除等危险操作

type: info 蓝色按钮，用于普通确认

默认 黄色图标，未指定 type 时

所有 admin 子页面的原生 confirm 已全部替换。

侧边栏与导航

侧边栏结构

分组（nav-group）可折叠，点击标题切换。事件委托处理导航点击，避免逐个绑定。

移动端：默认隐藏，通过汉堡按钮切换，点击遮罩层关闭。

桌面端：按钮切换显示/隐藏。

导航数据

通过 data-* 属性传递信息：

data-page → 页面 ID（对应 PAGE_MAP）

data-title → 标签标题

data-icon → FA 图标名

data-closable → 是否可关闭（0/1）

核心特性

零依赖

完全自研，不依赖 layui、jQuery、Bootstrap 等框架。使用原生 JS（fetch API、FormData、classList、event delegation）实现所有交互。

移动适配

响应式布局：侧边栏滑出覆盖，标签栏单指滑动，触摸事件优化（touch-action: manipulation），stat 卡片 2 列网格，弹窗全屏底部弹出。

自定义搜索下拉

自研搜索下拉组件，支持键盘导航（↑↓ + Enter）、输入过滤、点击关闭。用于添加域名页面的用户选择。

兼容层

compat.css 提供最小第三方类名等价样式，使未改造的子页面（如 order_list.html）仍可正常显示，无需逐页修改。
